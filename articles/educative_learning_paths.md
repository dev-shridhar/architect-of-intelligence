# Educated Learning Paths: System Design & Agentic AI 🎓📚

Educative.io offers structured, interactive, text-based courses. Below is a curated catalog of their top courses and paths designed for **System Architects** and **AI Engineers**.

---

## 🏗️ 1. System Design Learning Paths

While the historical *Grokking the System Design Interview* has migrated to DesignGurus.io, Educative offers several native alternatives:

### A. [Grokking System Design Interview: Patterns & Mock Interviews](https://www.educative.io/)
*   **Focus**: Standard distributed system patterns (caching, partition keys, indices) broken down through mock interview scenarios.
*   **Key Case Studies**: Designing TinyURL, YouTube, Uber, and WhatsApp architectures.

### B. [Fundamentals of System Design](https://www.educative.io/)
*   **Focus**: Theoretical and practical building blocks of system architecture.
*   **Key Topics**: CAP Theorem, database replication lag, load balancing algorithms, rate limiting, and CDNs.

### C. [Machine Learning System Design](https://www.educative.io/)
*   **Focus**: Architecture specifically for ML infrastructure.
*   **Key Topics**: Feature stores, offline/online training patterns, model serving optimizations, handling data drift, and metrics logging.

## 🤖 2. Agentic AI & LLM Systems Paths

For AI Engineers looking to move past simple API wrappers into stateful, autonomous systems, Educative has developed new paths:

### A. [Agentic AI Expert Path](https://www.educative.io/)
*   **Focus**: Complete learning path from simple LLM calls to autonomous agent networks.
*   **Key Modules**:
    *   **CrewAI Framework**: Building role-playing collaborative agents (e.g., Researcher + Editor collaborations).
    *   **Agentic Design Patterns**: Task decomposition, self-correction loops, and tool routing.
    *   **Scaling Agents via MCP**: Utilizing Model Context Protocol to bind agents with databases and environment APIs.

### B. [Mastering MCP: Building Advanced Agentic Applications](https://www.educative.io/)
*   **Focus**: Detailed study of the **Model Context Protocol (MCP)**, the open standard for connecting AI models to tools.
*   **Key Modules**:
    *   **MCP Architecture**: Developing Host, Client, and Server nodes.
    *   **Tool Integration**: Connecting LLM agents to databases, code execution sandboxes, and web scrapers.
    *   **Security & Guardrails**: Managing permissions and sandboxing execution outputs.

### C. [LangChain & LLM Integrations](https://www.educative.io/)
*   **Focus**: Building LLM-backed pipelines.
*   **Key Modules**:
    *   **Chains & Prompt Templates**: Formatting intermediate prompt templates dynamically.
    *   **Vector DB RAG**: Connecting LangChain to vector indices (Pinecone, Chroma) for query semantic searches.
